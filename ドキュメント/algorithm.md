# スケジューリングアルゴリズム

## 目的

「全員OK日」は存在しない前提で、**譲り合いを可視化する**ための材料を計算する。

## ハード制約

| ID | 内容 |
|---|---|
| H1 | 小澤候補帯内（`ozawa-range.json` で定義） |
| H2 | 他都市と同日重複禁止 |
| H3 | 連日研修は基本禁止、ただし以下は例外 |
| H3-ex1 | 浜松L1↔浜松L2 連日 OK |
| H3-ex2 | 浜松L1/L2↔静岡 連日 OK |
| H4 | 宇都宮 と 浜松/静岡 の連日は厳禁 |

## ソフト制約と重み（初期値、rules.json で調整可）

| ID | 内容 | 重み |
|---|---|---:|
| S1 | HL1↔HL2 連日 | +10 |
| S2 | 横浜↔宇都宮 同週 | +8 |
| S3 | 浜松↔静岡 同週 | +8 |
| S4 | 宇都宮→1日挟む→横浜 | +20（宝くじ） |
| S5 | 浜松↔静岡 連日 or 1日置き | +20（宝くじ） |
| S6 | 地理的近接訪問 | +5 |
| S7 | 他研修被り | -3/件 |
| S8 | 土日・祝日 | -5 |

## 総合スコア

```
Score(date, city) = w_ok × OK社数
                  + w_maybe × MAYBE社数
                  + w_ng × NG社数    (w_ng は負)
                  + Σ ソフトヒット
                  + 他研修ペナルティ
                  + 土日祝ペナルティ

ハード違反があれば -Infinity
```

## 最小譲歩レコメンド

1. 上位5日について、各社×各ステータス上げ（NG→MAYBE or MAYBE→OK or 未→OK）を1つだけ試行
2. 再ランク後の最上位スコアが上がる組み合わせを ΔScore 降順で列挙
3. 上位3件をサイドパネルに表示

```
「B社が△→○にすれば 8/18(火) が最有力に」
```

計算量：候補日数 × 会社数 × 状態数（ほぼ2）× 全月ランク再計算1回＝数千オペで<10ms。

## ISO週番号

同週判定は ISO-8601 の週番号（月曜始まり）を使用。月をまたぐ同週（例：8/31(月) と 9/4(金)）も同週扱い。
